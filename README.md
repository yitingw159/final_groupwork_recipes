# 美食菜谱应用

基于 Flutter 的跨平台美食菜谱应用，集菜谱浏览、智能搜索、烹饪计时、暗色模式、收藏管理、课程思政于一体，采用 Material Design 3 设计风格。

## 功能特性

### 🏠 首页 — 菜谱浏览与分类筛选
- 橙粉渐变风格头部，美观大方
- 横向滑动分类标签栏（带 emoji 前缀），支持按菜系分类筛选
- 彩色渐变菜谱卡片，展示菜名、分类、地区、烹饪时间、食材预览
- 下拉刷新，实时获取最新数据
- 每张卡片附带收藏按钮，一键收藏喜欢的菜谱

### 🔍 搜索 — 菜名与食材双重搜索
- **菜名搜索**：输入菜名关键词即可快速匹配
- **食材反搜**：输入食材名称，反向查找包含该食材的所有菜谱
- **热门食材标签栏**：提供鸡肉、豆腐、番茄、鸡蛋等 10 个热门食材快捷标签，点击即可触发食材反搜
- **搜索历史**：使用 SharedPreferences 本地持久化，最多保存 8 条搜索记录，支持一键清空
- **卡片式搜索结果**：展示菜名、分类标签、地区、烹饪时间、食材预览
- **空结果友好提示**：无匹配结果时显示引导界面，帮助用户重新搜索

### ❤️ 收藏 — 我的菜谱收藏夹
- 一键收藏/取消收藏喜欢的菜谱
- 收藏列表卡片式展示，点击可进入详情页
- 右侧删除按钮，方便管理收藏
- 空收藏状态友好提示

### 📋 菜谱详情 — 食材、做法、营养全览
- **Hero 头图区**：菜谱精美图片 + 菜名 + 分类/地区标签 + 收藏按钮
- **基础信息卡片**：分类、地区、烹饪用时一目了然
- **食材展示**：所有食材以圆角 Chip 形式排列
- **营养信息网格**：卡路里、蛋白质、脂肪、碳水四大指标，响应式布局
- **烹饪步骤时间轴**：分步展示做法，每步附带预计时间标签

### ⏱️ 烹饪计时器 — 步骤级精准计时
- 每个烹饪步骤自带计时按钮，自动从步骤文字中提取时长（如"5分钟")
- 弹出圆形倒计时对话框，显示 MM:SS 进度
- 支持**开始 / 暂停 / 重置 / 关闭**操作
- 计时结束后自动标记步骤完成，弹出 SnackBar 提示
- 底部固定按钮：动态显示"开始烹饪"→"继续烹饪：步骤 N"→"全部完成"
- 完成进度条 + "已完成 N / M" 提示，让烹饪过程有条不紊

### 🌙 暗色模式 — 深色主题一键切换
- Material Design 3 深色主题，以深橙色为主色调
- 关于与设置页提供 Switch 开关 + AppBar 右上角月亮/太阳图标快捷切换
- 主题偏好使用 SharedPreferences 持久化，重启应用自动恢复上次选择

### ⚙️ 关于与设置 — 应用信息与合规声明
- **应用信息卡片**：应用名、版本号、简介
- **暗色模式切换卡片**：一键切换深色/浅色主题
- **隐私政策**：详细说明数据收集、本地存储、网络请求、权限声明
- **开源合规声明**：声明参考来源与自主开发内容
- **数据来源说明**：远程加载策略与本地后备机制
- **小组分工表**：展示团队成员及各自任务
- **课程思政总结**：工匠精神、团队协作、技术自信、服务社会、总结展望

### 🌐 远程数据加载 — 多源容灾策略
- **移动端**优先从 Gitee Raw 加载菜谱数据
- **Web 端**优先从 GitHub Raw 加载菜谱数据
- 远程加载失败时自动回退到本地 `data/meals.json`，确保应用始终可用
- 加载状态动画 + 文字提示，用户体验友好

### 📱 跨平台支持
- 支持 Web、Android、iOS 多平台运行
- Material Design 3 + Material You 设计风格

## 技术栈

- Flutter SDK ^3.6.0
- Dart SDK ^3.6.0
- Provider (状态管理 — 菜谱数据 + 主题切换)
- Dio (网络请求 — 远程数据加载)
- SharedPreferences (本地持久化 — 主题偏好 + 搜索历史)

## 数据来源

应用采用**远程加载为主，本地后备为辅**的策略：

### 远程数据源

- **Web 平台**: GitHub Raw
  ```
  https://raw.githubusercontent.com/yitingw159/final_groupwork_recipes/main/data/meals.json
  ```

- **移动端**: Gitee Raw
  ```
  https://gitee.com/Yiting_world/recipes_data/raw/master/data/meals.json
  ```

### 本地后备数据

当远程加载失败时，应用会自动使用本地资源文件 `data/meals.json`。

## 项目结构

```
├── lib/
│   ├── main.dart              # 入口文件，MultiProvider + 主题配置 + 4 Tab导航
│   ├── models/                # 数据模型
│   │   └── meal.dart          # 菜谱模型
│   ├── pages/                 # 页面组件
│   │   ├── home_page.dart     # 首页（分类筛选 + 菜谱卡片列表）
│   │   ├── search_page.dart   # 搜索页（菜名/食材搜索 + 热门标签 + 搜索历史）
│   │   ├── favorites_page.dart # 收藏页（收藏列表 + 删除管理）
│   │   ├── meal_detail_page.dart # 菜谱详情页（食材 + 营养 + 步骤 + 烹饪计时器）
│   │   └── about_page.dart    # 关于与设置页（暗色模式 + 隐私政策 + 课程思政）
│   ├── providers/             # 状态管理
│   │   ├── meal_provider.dart # 菜谱数据 + 收藏状态管理
│   │   └── theme_provider.dart # 暗色模式主题状态管理（SharedPreferences持久化）
│   └── services/              # 服务层
│       └── data_service.dart  # 远程数据加载服务（Gitee/GitHub + 本地后备）
├── data/
│   └── meals.json             # 本地菜谱数据（10条）
├── images/                    # 运行截图
├── android/                   # Android 原生配置
├── ios/                       # iOS 原生配置
├── web/                       # Web 配置
└── pubspec.yaml               # 项目依赖配置
```
